# Sanity CMS Content Types

Use this as a quick map of what each Sanity Studio sidebar item controls.

| Studio label | Controls | Public pages affected |
|---|---|---|
| `Insight Article` | Editorial articles, explainers, market notes, and neighbourhood editorial content. | `/insights`, `/insights/[slug]`, related insight links on service pages, sitemap |
| `Case Study` | Client/project stories showing what HIDD prevented or uncovered. | `/case-studies`, `/case-studies/[slug]` |
| `Report Download` | Downloadable reports, checklists, gated PDFs/files, and coming-soon library assets. | `/reports`, `/insights` library surfaces, homepage featured download cards |
| `Risk Map District` | Risk map polygons, district labels, popup summaries, risk grades, layer scores, and district brief pages. | Homepage map, `/risk-map`, `/risk-map/[slug]` |
| `Social Proof` | Testimonials, client signals, and recent activity items. | Homepage social proof section |
| `FAQ` | Public questions and answers. | `/faqs`, homepage FAQ preview |

## Notes for editors

- Use `Insight Article` for written posts that should read like articles.
- Use `Report Download` for files or downloadable resources, even if the asset is a checklist rather than a formal report.
- Use `Risk Map District` when editing neighbourhood map data, district brief copy, or polygon coordinates.
- Do not rename slugs after publication unless you also want the public URL to change.
